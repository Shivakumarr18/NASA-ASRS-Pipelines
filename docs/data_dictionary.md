# 📖 Data Dictionary: Aviation Risk Triage Platform

This document serves as the **Architectural Blueprint** for the project. It maps the raw NASA ASRS multi-header CSV data into a flattened, high-performance SQL schema for the Gold Layer warehouse.

## 🏗️ Core Schema Mapping

| Category (Row 1) | Raw Header (Row 2)     | SQL Column Name  | Data Type | Architectural Logic / Rule                                           |
| :--------------- | :--------------------- | :--------------- | :-------- | :------------------------------------------------------------------- |
| **-**            | **ACN**                | `report_id`      | `INTEGER` | **Primary Key.** Unique identifier for the safety report.            |
| **Time**         | **Date**               | `incident_date`  | `DATE`    | String (YYYYMM) to be cast as a standardized Date object.            |
| **Place**        | **State Reference**    | `state_code`     | `STRING`  | Geographical context for regional risk heatmaps.                     |
| **Environment**  | **Flight Conditions**  | `weather_type`   | `STRING`  | VMC (Visual) vs IMC (Instrument). High weight in risk tiering.       |
| **Environment**  | **Light**              | `lighting_cond`  | `STRING`  | Daylight, Night, Dawn, Dusk. Critical for visibility analysis.       |
| **Aircraft 1**   | **Make Model Name**    | `aircraft_model` | `STRING`  | Normalized aircraft model (e.g., Boeing 737, Airbus A320).           |
| **Aircraft 1**   | **Flight Phase**       | `flight_phase`   | `STRING`  | Phase of flight (e.g., Takeoff, Landing). Critical for triage logic. |
| **Component**    | **Aircraft Component** | `component_id`   | `STRING`  | The specific mechanical part or system reported as failing.          |
| **Assessments**  | **Primary Problem**    | `issue_category` | `STRING`  | High-level categorization of the reported anomaly.                   |
| **Report 1**     | **Narrative**          | `raw_narrative`  | `TEXT`    | **AI INPUT.** The unstructured text processed by GPT-4o.             |

---

## 🛠️ Derived AI Columns (Gold Layer)

_These columns do not exist in the raw data but will be generated by our AI Enrichment Pipeline (Phase 3)._

| SQL Column Name  | Data Type | Logic                                                                  |
| :--------------- | :-------- | :--------------------------------------------------------------------- |
| `ai_risk_tier`   | `STRING`  | Automated classification (CRITICAL, HIGH, MEDIUM, LOW).                |
| `ai_risk_reason` | `TEXT`    | The "Why" behind the AI's classification for human audit.              |
| `is_recurring`   | `BOOLEAN` | Calculated flag if the same `component_id` appears in a 30-day window. |
